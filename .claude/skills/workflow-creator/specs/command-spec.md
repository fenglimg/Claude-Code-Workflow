## File Structure

### Directory Layout

```
.claude/commands/
├── {namespace}/                    # Command namespace (workflow, issue, memory, etc.)
│   ├── {command}.md               # Single command file
│   ├── {sub-namespace}/           # Nested namespace
│   │   └── {command}.md
│   └── tools/                     # Utility commands
│       └── {tool-command}.md
└── {standalone-command}.md        # Root-level command
```

### Naming Conventions

| Element | Convention | Example |
|---------|------------|---------|
| Namespace | lowercase, singular | `workflow`, `issue`, `memory` |
| Command file | lowercase, hyphenated | `lite-plan.md`, `test-gen.md` |
| Sub-namespace | lowercase, hyphenated | `ui-design`, `session` |
| Tool command | lowercase, hyphenated | `context-gather.md`, `conflict-resolution.md` |

### Command Path Resolution

```
/workflow:lite-plan     → .claude/commands/workflow/lite-plan.md
/issue:discover         → .claude/commands/issue/discover.md
/workflow:session:start → .claude/commands/workflow/session/start.md
/ccw                    → .claude/commands/ccw.md
```

## Front Matter Specification

### Required Fields

```yaml
---
name: {command-name}
description: {one-line description of command purpose and capabilities}
argument-hint: "{usage pattern with flags and arguments}"
allowed-tools: {comma-separated list of allowed tools}
---
```

### Field Definitions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Command identifier (matches filename without .md) |
| `description` | string | Yes | One-line description (50-150 chars) |
| `argument-hint` | string | Yes | Usage pattern showing flags and arguments |
| `allowed-tools` | string | Yes | Tools the command can use (with wildcards) |

### Argument Hint Syntax

```
"[-f|--flag] [-o|--option <value>] <required-arg> [optional-arg]"

Conventions:
- [-f|--flag]           Optional flag (short|long form)
- [-o|--option <value>] Optional flag with value
- <required-arg>        Required positional argument
- [optional-arg]        Optional positional argument
- "description"|file.md Alternative input formats
```

### Allowed Tools Patterns

| Pattern | Meaning | Example |
|---------|---------|---------|
| `Tool(*)` | Tool with any parameters | `Task(*)`, `Read(*)` |
| `Tool` | Tool without parameters | `Bash`, `Glob` |
| `mcp__server__*` | All MCP tools from server | `mcp__chrome__*` |
| Comma-separated | Multiple tools | `Task(*), Read(*), Bash` |

### Example Front Matter

```yaml
---
name: lite-plan
description: Lightweight interactive planning workflow with in-memory planning, code exploration, and execution execute to lite-execute after user confirmation
argument-hint: "[-y|--yes] [-e|--explore] \"task description\"|file.md"
allowed-tools: TodoWrite(*), Task(*), SlashCommand(*), AskUserQuestion(*)
---
```

## Command Body Structure

### Standard Sections

```markdown
# {Namespace} {Command-Name} Command (/{namespace}:{command-name})

## Overview
{2-3 paragraph description of command purpose, capabilities, and workflow}

## Usage
{Code block with usage examples}

## Output Artifacts
{Table of generated files and their descriptions}

## Execution Process
{ASCII flowchart or numbered phase list}

## Implementation
{Detailed phase-by-phase implementation with code examples}

## Session Folder Structure
{Directory tree of output artifacts}

## Error Handling
{Table of errors and resolutions}
```

### Section Requirements

| Section | Required | Purpose |
|---------|----------|---------|
| Overview | Yes | High-level command description |
| Usage | Yes | Command syntax and examples |
| Output Artifacts | Yes | Files generated by command |
| Execution Process | Yes | Workflow phases overview |
| Implementation | Yes | Detailed phase implementation |
| Session Folder Structure | Recommended | Output directory layout |
| Error Handling | Recommended | Error recovery strategies |

## Flag Conventions

### Standard Flags

| Flag | Short | Purpose | Default Behavior |
|------|-------|---------|------------------|
| `--yes` | `-y` | Skip confirmations | Interactive mode |
| `--explore` | `-e` | Force exploration phase | Auto-detect |
| `--hotfix` | `-h` | Production hotfix mode | Normal mode |
| `--in-memory` | - | Use in-memory context | File-based context |
| `--mode` | `-m` | Execution mode | Default mode |

### Flag Parsing Pattern

```javascript
// Standard flag parsing
const autoYes = $ARGUMENTS.includes('--yes') || $ARGUMENTS.includes('-y')
const forceExplore = $ARGUMENTS.includes('--explore') || $ARGUMENTS.includes('-e')
const hotfixMode = $ARGUMENTS.includes('--hotfix') || $ARGUMENTS.includes('-h')

// Extract positional arguments (after removing flags)
const positionalArgs = $ARGUMENTS
  .filter(arg => !arg.startsWith('-'))
  .join(' ')
```

## Session Management

### Session ID Generation

```javascript
// Standard session ID format: {slug}-{YYYY-MM-DD}
const getUtc8ISOString = () => new Date(Date.now() + 8 * 60 * 60 * 1000).toISOString()

const taskSlug = task_description
  .toLowerCase()
  .replace(/[^a-z0-9]+/g, '-')
  .substring(0, 40)
const dateStr = getUtc8ISOString().substring(0, 10)

const sessionId = `${taskSlug}-${dateStr}`
const sessionFolder = `.workflow/.{command-type}/${sessionId}`
```

### Session Folder Patterns

| Command Type | Folder Pattern | Example |
|--------------|----------------|---------|
| lite-plan | `.workflow/.lite-plan/{slug}-{date}/` | `.workflow/.lite-plan/implement-jwt-2025-11-29/` |
| lite-fix | `.workflow/.lite-fix/{slug}-{date}/` | `.workflow/.lite-fix/login-fails-2025-11-29/` |
| brainstorm | `.workflow/.brainstorm/BS-{topic}-{date}/` | `.workflow/.brainstorm/BS-auth-refactor-2025-11-29/` |

## Agent Invocation

### Task Tool Pattern

```javascript
Task(
  subagent_type="{agent-name}",
  run_in_background=false,  // MANDATORY: Wait for results
  description="{brief description}",
  prompt=`
## Task Objective
{Clear objective statement}

## Output Location
**Session Folder**: ${sessionFolder}
**Output File**: ${sessionFolder}/{output-file}

## Assigned Context
{Context variables passed to agent}

## MANDATORY FIRST STEPS
{Steps agent must execute first}

## Expected Output
{Output format and schema reference}

## Success Criteria
{Checklist of completion criteria}

## Execution
**Write**: \`${sessionFolder}/{output-file}\`
**Return**: {Brief summary format}
`
)
```

### SlashCommand Delegation

```javascript
// Delegate to another command with context
SlashCommand(command="/{namespace}:{command} {flags}")

// Example: Delegate to lite-execute
SlashCommand(command="/workflow:lite-execute --in-memory")
```

## User Interaction

### AskUserQuestion Pattern

```javascript
AskUserQuestion({
  questions: [
    {
      question: "{Question text}",
      header: "{Short header (max 12 chars)}",
      multiSelect: false,
      options: [
        { label: "{Option 1}", description: "{Description}" },
        { label: "{Option 2}", description: "{Description}" },
        // Max 4 options per question
      ]
    }
  ]
})
```

### Multi-Round Clarification

```javascript
// AskUserQuestion limits: max 4 questions per call
const BATCH_SIZE = 4
const totalRounds = Math.ceil(questions.length / BATCH_SIZE)

for (let i = 0; i < questions.length; i += BATCH_SIZE) {
  const batch = questions.slice(i, i + BATCH_SIZE)
  const currentRound = Math.floor(i / BATCH_SIZE) + 1
  
  console.log(`### Clarification Round ${currentRound}/${totalRounds}`)
  AskUserQuestion({ questions: batch })
}
```

## Validation Checklist

### Command File Validation

- [ ] **Front matter complete**: All required fields present
- [ ] **Name matches filename**: `name` field equals filename without .md
- [ ] **Description is concise**: 50-150 characters
- [ ] **Argument hint accurate**: Reflects actual flags and arguments
- [ ] **Allowed tools specified**: All used tools listed

### Content Validation

- [ ] **Overview section**: Clear purpose and capabilities
- [ ] **Usage examples**: At least 2-3 usage examples
- [ ] **Output artifacts table**: All generated files documented
- [ ] **Execution process**: Clear phase breakdown
- [ ] **Implementation details**: Code examples for each phase
- [ ] **Error handling**: Common errors and resolutions

### Integration Validation

- [ ] **Session folder created**: mkdir -p with verification
- [ ] **Agent invocations correct**: run_in_background=false
- [ ] **Schema references valid**: Paths to schema files correct
- [ ] **Delegation targets exist**: SlashCommand targets are valid
